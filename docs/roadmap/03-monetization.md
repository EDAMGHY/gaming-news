# Phase 3: Monetization

## Goal

Add revenue features after the MVP has stable content, search, SEO, and enough traffic to justify monetization work.

Do not prioritize monetization before the site feels trustworthy and useful.

## 1. Best Early Monetization Paths

- [ ] Affiliate links on game pages.
- [ ] Affiliate links on review pages.
- [ ] Buying guides with affiliate links.
- [ ] Newsletter sponsorships after newsletter growth.
- [ ] Display ads after traffic is meaningful.

## 2. Affiliate System

- [ ] Add external store links to games.
- [ ] Store fields can include platform/store name, URL, price label, and affiliate disclosure.
- [ ] Add `Where to buy` section on game pages.
- [ ] Add affiliate disclosure text globally or near affiliate modules.
- [ ] Add affiliate links to reviews when relevant.
- [ ] Track clicks if analytics policy allows it.

## 3. Deals Section

- [ ] Add `/deals` route later.
- [ ] Decide if deals should be an article category or separate collection.
- [ ] Add fields for sale price, original price, discount, store, platform, and expiry date.
- [ ] Add homepage deals block.
- [ ] Add deal expiry behavior so old deals do not stay prominent.

## 4. Sponsored Content

- [ ] Add sponsored flag to articles if sponsored posts are planned.
- [ ] Add clear sponsored disclosure on public pages.
- [ ] Add sponsor name field.
- [ ] Keep sponsored content visually distinct enough to maintain trust.
- [ ] Do not mix sponsored labels with normal editorial reviews.

## 5. Display Ads

- [ ] Add ad slots only after the layout is stable.
- [ ] Suggested ad slots: below hero, between archive cards, in article body, sidebar on desktop.
- [ ] Avoid intrusive ads on mobile.
- [ ] Keep Core Web Vitals in mind.
- [ ] Use placeholder components first before integrating an ad network.

## 6. Newsletter Monetization

- [ ] Build newsletter signup first in Phase 2.
- [ ] Add sponsor slot in newsletter later.
- [ ] Add weekly digest format.
- [ ] Promote best reviews, guides, deals, and upcoming releases.

## 7. Premium Or Membership Ideas

- [ ] Members-only buying guides.
- [ ] Ad-free experience.
- [ ] Early access to long-form features.
- [ ] Community features.
- [ ] Only consider this after audience demand exists.

## 8. Monetization Acceptance Criteria

- [ ] Monetization is clearly disclosed.
- [ ] Ads or affiliate modules do not break mobile UX.
- [ ] Revenue features do not make editorial content less trustworthy.
- [ ] The site remains fast enough after third-party scripts are added.
